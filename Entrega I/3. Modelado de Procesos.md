# **3\. Modelado de Procesos (SBPMN)**

Se presentan los diagramas del proceso de negocio simulando el estándar BPMN con "Swimlanes" (Carriles) para identificar las responsabilidades del Cliente, el Sistema, la Automatización y el Agente.

---

## **3.1 Flujo Semestre I (Alcance Actual)**

En el Semestre I el sistema opera **sin IA ni n8n**. La clasificación se realiza únicamente con motor de reglas (palabras clave).

```mermaid
flowchart TD
    subgraph Cliente
        A((Inicio)) --> B[Inicia sesión en el portal]
        B --> C[Redacta y envía Ticket de Soporte]
        O[Recibe solución y confirmación] --> P((Fin))
    end

    subgraph Sistema_Sem1
        C --> D[Recibe payload en FastAPI]
        D --> F[Aplica Motor de Reglas / Palabras Clave]
        F --> I[Almacena en Base de Datos Supabase]
    end

    subgraph Agente_Soporte
        I --> M[Accede al Dashboard]
        M --> N[Gestiona y cierra el Ticket]
        N --> O
    end

    classDef startEnd fill:#d4edda,stroke:#28a745,stroke-width:2px
    classDef process fill:#e2e3e5,stroke:#6c757d,stroke-width:1px
    class A,P startEnd
    class B,C,D,F,I,M,N,O process
```

---

## **3.2 Flujo Proyecto Completo (Semestres 2 y 3)**

Cuando se integren IA y n8n, el flujo evoluciona con decisión de fase IA y webhooks de alerta.

```mermaid
flowchart TD
    subgraph Cliente
        A((Inicio)) --> B[Inicia sesión en el portal]
        B --> C[Redacta y envía Ticket de Soporte]
        O[Recibe solución y confirmación] --> P((Fin))
    end

    subgraph Sistema_AI_CoPilot
        C --> D[Recibe payload en FastAPI]
        D --> E{¿Fase de IA Activa?}
        E -->|No| F[Aplica Motor de Reglas/Palabras Clave]
        E -->|Sí| G[Inferencia con LLM Llama-3.1]
        G --> H[Extrae Categoría y Sentimiento]
        F --> I[Almacena en Base de Datos Supabase]
        H --> I
    end

    subgraph Automatizacion_n8n
        I --> J{¿Sentimiento Negativo/Urgente?}
        J -->|Sí| K[Dispara Webhook de Alerta a Telegram/Email]
        J -->|No| L[Flujo estándar]
    end

    subgraph Agente_Soporte
        K --> M[Recibe alerta e ingresa al Dashboard]
        L --> M
        M --> N[Gestiona, resuelve y cierra el Ticket]
        N --> O
    end

    classDef startEnd fill:#d4edda,stroke:#28a745,stroke-width:2px
    classDef gateway fill:#fff3cd,stroke:#ffc107,stroke-width:2px
    classDef process fill:#e2e3e5,stroke:#6c757d,stroke-width:1px
    class A,P startEnd
    class E,J gateway
    class B,C,D,F,G,H,I,K,L,M,N,O process
```
