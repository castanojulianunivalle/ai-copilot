# **2\. Definición de la Arquitectura y Stack Tecnológico**

## **2.1 Stack Tecnológico Definido**

El proyecto se construye sobre una arquitectura de microservicios y tecnologías modernas para garantizar escalabilidad y separación de responsabilidades:

* **Frontend:** React 18, Vite, Tailwind CSS, Framer Motion (Dashboard SPA con actualizaciones en tiempo real).  
* **Backend:** FastAPI (Python), Pydantic, Uvicorn (API REST asíncrona).  
* **Base de Datos & Auth:** Supabase (PostgreSQL) con Row Level Security (RLS) y Supabase Realtime.  
* **Inteligencia Artificial (NLP):** Llama-3.1-8B-Instruct consumido a través de Hugging Face Router / vLLM (Compatible con API OpenAI).  
* **Automatización:** n8n (Motor de flujos basado en nodos) para integración con Telegram y Email.  
* **Infraestructura:** Docker Compose (Orquestación local de contenedores).

## **2.2 Componente Inteligente**

El núcleo de inteligencia radica en la capa de Backend (FastAPI). En lugar de usar complejas expresiones regulares o algoritmos de machine learning tradicional, el sistema delega la extracción de entidades y el análisis de sentimiento a un LLM (Llama-3.1). Mediante *Prompt Engineering* estricto, el modelo devuelve un JSON estructurado con la "Categoría" exacta del problema y el "Sentimiento" (Positivo, Neutral, Negativo, Urgente) del usuario.
