## **6.3 Release Plan (Plan de Entregas)**

### **Release 1: Arquitectura Base y Gestión Core**

| Atributo | Detalle |
|----------|---------|
| **Periodo** | Sprint 0 + Sprint 1, 2, 3 / Semestre 1 |
| **Sprints** | Sprint 0 (2 sem), Sprint 1–3 (3 sem cada uno) |
| **Objetivo** | Sistema funcional transaccional sin IA |
| **Contenido** | HU-01 (Registro), HU-02 (Login), HU-03 (Crear Ticket), HU-04 (Dashboard y Gestión), HU-04b (Clasificación por reglas) |
| **Hito** | Aplicación web desplegada y Base de Datos configurada con Row Level Security |

**Detalle por Sprint:**
- **Sprint 0 (2 sem):** Kickoff, definición de entorno, repositorio, Jira y documentación inicial
- **Sprint 1 (3 sem):** Configuración de Supabase (Tablas, RLS), Setup de FastAPI + React e implementación de Auth y Perfiles de Usuario
- **Sprint 2 (3 sem):** Panel de creación y listado de tickets para el Cliente
- **Sprint 3 (3 sem):** Dashboard del Agente y motor de reglas de clasificación básico

## **6.4 Evidencia de Prácticas Ágiles (Scrum)**

Dado que el proyecto se desarrolla en modalidad *Solo Developer* (el estudiante asume Product Owner, Scrum Master y Developer), las ceremonias se adaptan a auto-gestión y revisión académica:

| Ceremonia | Ejecución | Evidencia |
|-----------|-----------|-----------|
| **Sprint Planning** | Primer día de cada Sprint (3 semanas): revisión del Backlog en Jira, selección de HUs por Story Points, movimiento a "To Do" | Sprints iniciados con metas en Jira |
| **Daily Scrum (Adaptado)** | Bitácora de commits en GitHub que reflejen avance diario y bloqueos | Historial de commits con mensajes descriptivos |
| **Sprint Review y Retrospective** | Demostración del incremento al Profesor/Asesor al final de cada iteración (y formalmente al final de semestre) | Documento de avances semestrales y actas de reunión |
