# **3\. Modelado de Procesos (SBPMN)**

Se presentan los diagramas del proceso de negocio simulando el estándar BPMN con "Swimlanes" (Carriles) para identificar las responsabilidades del Cliente, el Sistema, la Automatización y el Agente.

---

## **3.1 Flujo Semestre I (Alcance Actual)**

En el Semestre I el sistema opera **sin IA ni n8n**. La clasificación se realiza únicamente con motor de reglas (palabras clave).

```mermaid
flowchart TD
    subgraph Cliente
        A((Inicio)) --> B[Inicia sesion en el portal]
        B --> C[Redacta y envia Ticket]
        O[Recibe solucion] --> P((Fin))
    end

    subgraph Sistema
        C --> D[Recibe payload en FastAPI]
        D --> F[Aplica Motor de Reglas]
        F --> I[Almacena en Supabase]
    end

    subgraph Agente
        I --> M[Accede al Dashboard]
        M --> N[Gestiona y cierra Ticket]
        N --> O
    end

    classDef startEnd fill:#d4edda,stroke:#28a745
    classDef process fill:#e2e3e5,stroke:#6c757d
    class A,P startEnd
    class B,C,D,F,I,M,N,O process
```

---

## **3.2 Flujo Proyecto Completo (Semestres 2 y 3)**

Cuando se integren IA y n8n, el flujo evoluciona con decisión de fase IA y webhooks de alerta.

```mermaid
flowchart TD
    subgraph Cliente
        A((Inicio)) --> B[Inicia sesion en el portal]
        B --> C[Redacta y envia Ticket]
        O[Recibe solucion] --> P((Fin))
    end

    subgraph Sistema
        C --> D[Recibe payload en FastAPI]
        D --> E{Fase IA activa?}
        E -->|No| F[Aplica Motor de Reglas]
        E -->|Si| G[Inferencia con LLM Llama]
        G --> H[Extrae Categoria y Sentimiento]
        F --> I[Almacena en Supabase]
        H --> I
    end

    subgraph Automatizacion
        I --> J{Sentimiento negativo?}
        J -->|Si| K[Webhook Alerta Telegram o Email]
        J -->|No| L[Flujo estandar]
    end

    subgraph Agente
        K --> M[Recibe alerta y accede al Dashboard]
        L --> M
        M --> N[Gestiona y cierra Ticket]
        N --> O
    end

    classDef startEnd fill:#d4edda,stroke:#28a745
    classDef gateway fill:#fff3cd,stroke:#ffc107
    classDef process fill:#e2e3e5,stroke:#6c757d
    class A,P startEnd
    class E,J gateway
    class B,C,D,F,G,H,I,K,L,M,N,O process
```
