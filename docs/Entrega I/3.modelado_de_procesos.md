# **3\. Modelado de Procesos (SBPMN)**

Se presentan los diagramas del proceso de negocio simulando el estándar BPMN con "Swimlanes" (Carriles) para identificar las responsabilidades del Cliente, el Sistema, la Automatización y el Agente.

Los diagramas usan **flujo vertical** (`flowchart TB`) con **carriles estrechos** (pocas columnas) para ocupar **menos ancho** en diapositiva y mantener altura contenida.

---

## **3.1 Flujo Semestre I (Alcance Actual)**

En el Semestre I el sistema opera **sin IA ni n8n**. La clasificación se realiza únicamente con motor de reglas (palabras clave).

```mermaid
%%{init: {'flowchart': {'nodeSpacing': 10, 'rankSpacing': 14}, 'themeVariables': {'fontSize': '11px'}}}%%
flowchart TB
    subgraph CL["Cliente"]
        A((Inicio)) --> B[Inicia sesion]
        B --> C[Redacta y envia ticket]
    end

    subgraph S1["Sistema"]
        D[FastAPI recibe]
    end

    subgraph IA["Componente Inteligente"]
        F[Motor de reglas]
    end

    subgraph S2["Sistema"]
        I[Almacena Supabase]
    end

    subgraph AG["Agente"]
        M[Dashboard] --> N[Gestiona y cierra]
    end

    subgraph CL2["Cliente"]
        O[Recibe solucion] --> P((Fin))
    end

    C --> D --> F --> I --> M
    N --> O

    classDef startEnd fill:#d4edda,stroke:#28a745
    classDef process fill:#e2e3e5,stroke:#6c757d
    classDef intelligent fill:#e8dff7,stroke:#6f42c1
    class A,P startEnd
    class B,C,D,I,M,N,O process
    class F intelligent
    style IA fill:#e8dff7,stroke:#6f42c1
```

## **3.2 Flujo Proyecto Completo (Semestres 2 y 3)**

Cuando se integren IA y n8n, el flujo evoluciona con decisión de fase IA y webhooks de alerta.

```mermaid
%%{init: {'flowchart': {'nodeSpacing': 10, 'rankSpacing': 14}, 'themeVariables': {'fontSize': '11px'}}}%%
flowchart TB
    subgraph CL["Cliente"]
        A((Inicio)) --> B[Inicia sesion]
        B --> C[Redacta y envia ticket]
    end

    subgraph S1["Sistema"]
        D[FastAPI recibe]
    end

    subgraph IA["Componente Inteligente"]
        E{IA activa?}
        E -->|No| F[Reglas fallback]
        E -->|Si| G[LLM Llama-3.1]
        G --> H[Cat. y sentimiento]
    end

    subgraph S2["Sistema"]
        I[Almacena Supabase]
    end

    subgraph AU["Automatizacion"]
        J{Sentimiento -?}
        J -->|Si| K[Webhook TG / mail]
        J -->|No| L[Flujo estandar]
    end

    subgraph AG["Agente"]
        M[Alerta / Dashboard]
        M --> N[Gestiona y cierra]
    end

    subgraph CL2["Cliente"]
        O[Recibe solucion] --> P((Fin))
    end

    C --> D --> E
    F --> I
    H --> I
    I --> J
    K --> M
    L --> M
    N --> O

    classDef startEnd fill:#d4edda,stroke:#28a745
    classDef gateway fill:#fff3cd,stroke:#ffc107
    classDef process fill:#e2e3e5,stroke:#6c757d
    classDef intelligent fill:#e8dff7,stroke:#6f42c1
    class A,P startEnd
    class E,J gateway
    class B,C,D,I,K,L,M,N,O process
    class F,G,H intelligent
    style IA fill:#e8dff7,stroke:#6f42c1
```

### Nota para presentaciones

- **Ancho:** al ir **en columna** (TB), el diagrama suele ocupar **la mitad de ancho** respecto a un LR largo; ajusta zoom en la diapositiva al 80–100 % del alto útil.
- **Exportar:** SVG desde Mermaid Live o desde VS Code; en PowerPoint, **encajar** al marco sin estirar el ancho.
- Si **3.2** se ve alto, coloca **3.1** en una slide y **3.2** en la siguiente.
